# 運用方針

本リポジトリでは、業務フローをAIが理解しやすい形式（SKILL.md）で管理します。

## 管理の仕組み
- `sources/` にある原稿を正（Single Source of Truth）とします。
- 修正が必要な場合は、まず `sources/` 内のファイルを修正し、それを基に各出力を更新します。

## ブランチ戦略
- `main`: 常に本番利用可能な最新の状態を維持します。
- `feature/*`: 新機能の追加や修正を行う際に使用します。

## レビューフロー
- プルリクエストを作成し、内容の整合性を確認した後に `main` へマージします。
