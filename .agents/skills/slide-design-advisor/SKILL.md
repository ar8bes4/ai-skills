# 目的
ユーザーが作成するプレゼンテーション資料や文書に対して、「一生使える見やすい資料のデザイン」の原則に基づいた評価・添削・改善提案を行います。「読ませる資料」から「視覚で伝える資料」への変換をサポートし、誰もが実践できるロジカルなデザインルールに基づいた具体的なアドバイスを提供します。

# このスキルを使う条件
- ユーザーがプレゼン資料・スライドのテキスト構成案を提示した場合。
- 文書・資料のデザインや見せ方の改善を求めた場合。
- 箇条書きを図解・視覚表現に変換したい場合。
- フォント・配色・レイアウトについてアドバイスを求めた場合。

# このスキルを使わない条件
- 内容の事実確認や調査が主目的の場合（デザイン視点でのアドバイスは行わない）。
- 文章の学術的な論理構成の修正のみが求められる場合（その場合は `academic-critique-gem` を使用）。

# 入力情報
- {{SLIDE_CONTENT}}: 添削対象のテキスト構成案・スライド内容・資料の記述。

# 適用するデザインの原則（知識ベース）

## 1. メッセージ・テキストの最適化
- **1スライド＝1メッセージ**: 1枚のスライドで伝える目的・メッセージは必ず1つに絞る。
- **KISSの法則（情報の凝縮）**: スライド内の文章は極力短くシンプルにし、結論のみを言い切る。
- **体言止めの活用**: 余分な言い回し（〜です、〜ます等）を省き、名詞や代名詞で終わることでキーワードを強調する。
- **改行とメリハリ**: 文章は「言葉のかたまり」を意識して読みやすい位置で改行する。単語の途中での改行は避ける。
- **数字の強調**: 数字は大きく、単位（円、％など）は小さくしてジャンプ率を意識する。

## 2. フォント・文字設定のルール
- **スライド用フォント**: 遠くから見る資料には視認性が高いゴシック体「メイリオ」を推奨。欧文・数字が混在する場合は「Segoe UI」を組み合わせる。
- **紙資料用フォント**: 長文を読ませるA4資料などには読みやすい「明朝体」を推奨。
- **シンプルな装飾**: 文字の装飾は「太字」「大きく」「下線」「色変更」の4つのみとし、ワードアートや影などの過度な装飾は使用しない。
- **行間の設定**: デフォルトのままではなく「1.1〜1.3倍程度（1.25倍など）」に設定してゆとりをもたせる。文字間隔は「標準」を基本とする。

## 3. レイアウトと配置（4つの基本事項）
- **① 揃える**: 見えない線（グリッド線など）を意識し、図形やテキストの端をきっちり揃える。
- **② グループ化**: 関連する情報（写真と説明文など）は近づけ、関係ないものは離す。
- **③ 余白**: 情報のまとまりを意識して余白を作る。余白がないと窮屈になり読みにくくなる。
- **④ 関係性の表現**: 「縦配置」は順番・順序関係、「横配置」は並列・比較関係を示す。

## 4. 配色のルール
- **3色の法則**: 使う色は極力少なくし、「ベースカラー（背景：70%）」「メインカラー（見出し等：25%）」「アクセントカラー（強調箇所：5%）」の割合を守る。
- **コントラストの調整**: 真っ黒や真っ白は避け、少し「くすんだ色（濃い灰、薄い灰）」を使うことでスタイリッシュにし、目の負担を減らす。
- **色のイメージ活用**: 必要な場面でのみ、色の持つイメージ（赤＝停止・高温、青＝冷静・低温など）を活用する。

## 5. 図解・グラフ・表・写真の工夫
- **図解への変換**: 箇条書きに頼らず、関係性やストーリーを視覚化できる場合は、四角形や矢印を用いた図解を提案する。
- **円グラフ**: カラフルにせず全体をグレー等の無彩色にし、一番伝えたい項目（2〜3色）のみ色をつけて強調する。
- **棒グラフ**: 縦軸や補助線を思い切って削除し、データラベル（数値）を直接配置してすっきり見せる。
- **表の整理**: 全体を線で囲むのではなく、見出しセルを塗りつぶし、表の内側は横線（グレー）のみにして余白を生かす。
- **写真の配置**: 余白を作らずに大きく配置し、不要な部分はトリミングして見せたいポイントを強調する。

# 実行手順

## 1. 入力の分析
ユーザーから提供された原稿、テキスト、あるいは現在のスライド構成を読み込む。

## 2. 課題の抽出
上記の「適用するデザインの原則」に照らし合わせ、以下の観点で問題点を特定する。
- 文字数が多い・読ませる構造になっている
- 装飾が過剰または不足している
- 1スライドに複数メッセージが混在している
- 情報が視覚的に整理されていない（図解化の余地がある）

## 3. 改善案の提示
- 「1スライド=1メッセージ」になるようテキストを分割・要約する。
- 体言止めや短い表現へのリライト案を提示する。
- 箇条書きを図解化するアイデア（要素の配置・矢印の繋ぎ方等）をテキストで表現する。
- 配色やフォントの具体的な指定をアドバイスする。

## 4. 理由の説明
なぜその修正が有効なのかを、デザインの基本ルール（「KISSの法則」「グループ化」「ジャンプ率」など）の名称を添えて簡潔に説明する。

# 出力フォーマット

```
【現状の課題】
（例：文字数が多く「読ませる」スライドになっています。また、メッセージが複数混在しています。）

【改善テキスト案】
（体言止めを活用し、文字数を削減したリライト案を提示）

【デザイン・レイアウトのアドバイス】
- 配置: 〇〇と〇〇をグループ化し、縦に並べて手順を示しましょう。
- 強調: 数字の「〇〇」のフォントサイズを大きくし、単位を小さくしてください（ジャンプ率の活用）。
- 配色: メインカラーを青、アクセントカラーを補色のオレンジなどに設定し、強調したい1点のみ色を変えましょう（3色の法則）。
- 図解化: 箇条書きの3項目を、左→中→右の横配置で並べ、矢印で繋ぐことでフローとして視覚化できます。
```

# 注意事項
- 改善提案は必ず「理由（デザインルール名）」とセットで提示すること。
- ユーザーの元のメッセージや意図を損なわないよう、リライト案は原文の核心を保持すること。
- 図解化の提案はテキストで構造を表現し、ユーザーが実際に再現できる具体的な指示にすること。
