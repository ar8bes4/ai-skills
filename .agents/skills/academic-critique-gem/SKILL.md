# 目的
あらゆる提出テキストに対し、一切の妥協を排除した「建設的な批判」と、学術的に洗練された「リライト」を実行します。

# この skill を使う条件
- 文章の論理的な脆弱性を特定し、強化したい場合。
- 専門性の高い文書の構成や明瞭性を向上させたい場合。

# この skill を使わない条件
- 簡潔な要約のみが求められる場合。
- 事実確認を伴わない主観的なエッセイの校正。

# 入力情報
- {{TEXT}}: 分析・修正の対象となるテキスト。

# 実行手順

## 1. 事前確認
`<analysis>` タグ内にて、指定された4つの評価基準（論理、根拠、事実、構造）に沿って問題点を抽出します。

## 2. 情報整理
抽出した問題点を深刻度の高い順（論理的飛躍 > 根拠不足 > 表現の冗長性など）に整理します。

## 3. 実行
分析結果に基づき、徹底批判（Critique）と修正案（Improved Version）を構築します。

## 4. 点検
リライト後の文章が、元のテキストの「核となる意図」を損なっていないか最終確認します。

# 出力物
- 徹底批判（Critique）
- 修正案（Improved Version）
- 修正のポイント（Changes Made）
