# 目的
- ユーザーがアップロードしたGoogleスライドやPowerPointの内容を分析し、各スライドに最適なスピーカーノートを作成します。
- 聴衆の属性（例：専門家、学生、ビジネスパートナー）や、1スライドあたりの希望時間・分量に合わせて内容を調整します。
- ユーザーがコピー＆ペーストしやすいよう、各スライドのノートを個別のコードブロック形式で出力します。

# この skill を使う条件
- スライド資料が提供され、その内容に基づいた口頭発表の準備を行う場合。
- 特定の聴衆に向けて、メッセージの伝え方をカスタマイズしたい場合。

# この skill を使わない条件
- プレゼンテーション以外の用途（ブログ記事作成等）。
- 短い箇条書き程度のメモ作成。

# 入力情報
- {{PRESENTATION_DATA}}: スライドの内容
- {{AUDIENCE_PROFILE}}: 聴衆の属性
- {{TIME_CONSTRAINTS}}: 時間または分量の指定
- {{DESIRED_TONE}}: 全体的な雰囲気

# 実行手順

## 1. 初期ヒアリング
ユーザーに対し、以下の3点を確認します。
- プレゼンテーションの対象となる聴衆（ターゲット）。
- 1スライドあたりに充てる時間、または希望する文字数の目安。
- プレゼンテーションの全体的なトーン（例：フォーマル、カジュアル、情熱的）。

## 2. 分析と出力
- スライドの内容（テキストや画像の説明など）に基づき、自然で説得力のあるスクリプトを作成します。
- 専門用語の解説を入れるか、あるいは平易な言葉に言い換えるかなど、聴衆のレベルに合わせた表現を選択します。
- 各スライドの区切りを明確にし、以下の形式で出力します。

[スライド 1]
```
(ここにスピーカーノートの内容)
```

## 3. 点検
- トーンがプロフェッショナルかつ論理的であるか確認します。
- プレゼンターが自信を持って話せるような、リズムの良い文章になっているか最終確認します。

# 出力物
- スライド別スピーカーノートセット（[スライド X] ＋ コードブロック形式）
