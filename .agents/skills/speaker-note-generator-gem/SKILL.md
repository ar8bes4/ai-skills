# 目的
プレゼンテーションの成功を支援するため、スライドの内容から最も効果的なスピーカーノートを構築します。

# この skill を使う条件
- スライド資料が提供され、その内容に基づいた口頭発表の準備を行う場合。
- 特定の聴衆に向けて、メッセージの伝え方をカスタマイズしたい場合。

# この skill を使わない条件
- プレゼンテーション以外の用途（ブログ記事作成等）。
- 短い箇条書き程度のメモ作成。

# 入力情報
- {{PRESENTATION_DATA}}: スライドの内容
- {{AUDIENCE_PROFILE}}: 聴衆の属性
- {{TIME_CONSTRAINTS}}: 時間または分量の指定
- {{DESIRED_TONE}}: 全体的な雰囲気

# 実行手順

## 1. 初期ヒアリング
ユーザーに対し、聴衆（ターゲット）、1スライドあたりの目安（時間/文字数）、全体的なトーンを確認します。

## 2. 分析フェーズ
提供された各スライドのテキストや視覚情報をもとに、説得力のある論理構成を分析します。

## 3. 制作
聴衆のレベルに合わせた表現を選択し、各スライドのノートを独立したコードブロック形式で出力します。

## 4. 点検
トーンの一貫性と、プレゼンターが自信を持って話せる文章のリズムを確認します。

# 出力物
- スライド別スピーカーノートセット（[スライド 1] ``` ... ``` 形式）

# 参照資料
- なし

# 注意事項
プレゼンターの協力的なパートナーとして、自信を後押しするリズムの良い文章を心がけます。
