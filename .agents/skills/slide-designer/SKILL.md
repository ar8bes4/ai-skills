このワークフローは、ディープリサーチから得られた大量の情報を「1スライド1メッセージ」の黄金律に従って整理し、視覚的に訴求力の高いプレゼンテーション資料（Google Slides）を最短で作成するための手順です。

# 制作プロセス

### ステップ①：テーマのリサーチ（ディープリサーチ）
- **Antigravityの支援**: `search_web` や `read_url_content` を活用し、指定されたテーマに関する最新のエビデンス、データ、トレンドを収集します。
- **目的**: 信頼性の高い一次情報を確保すること。

### ステップ②：スライド構成案のドラフト作成
- **Antigravityの支援**: `slide-designer` スキルを活用し、リサーチ結果をスライドの「1枚1枚の構成」に分解します。
- **アウトプット**: 
    - ターゲット（学会用/患者用）に合わせたトーンの決定。
    - 1枚ごとのタイトルと、伝えたい唯一のメッセージ。
    - 図解（シェーマ）のアドバイス。

### ステップ③：Gemini Canvas への投入とスライド変換
- **外部ツール**: Gemini の Canvas 機能（または UI に直接）構造化された Markdown テキストを貼り付けます。
- **操作指示**: 「この内容に基づき、Google Slides の構成案を作成してください」と Gemini に指示。
- **ヒント**: 日本語のレイアウトが崩れないよう、文字数を絞ったドラフトを優先させます。

### ステップ④：Google Slides での編集とデザイン調整
- **外部ツール**: 生成された Google Slides を開き、全体のレイアウト、フォント、色（ベース・メイン・アクセント）を調整します。
- **重要点**: 箇条書きを 5〜7 行以内に抑え、24pt 以上の可読性を確保します。

### ステップ⑤：AI による最終検品と美装のアドバイス
- **Antigravityの支援**: 編集中のスライドのスクリーンショットやテキストを `slide-designer` に読み込ませ、「見栄えを良くする」ための追加アドバイスを求めます。
- **内容例**: 「このグラフの凡例を消して、アクセントカラーを右下の結論に置いてください」といった具体的な指示。

# 推奨設定
- **フォント**: メイリオ（Windows）/ ヒラギノ角ゴシック（Mac）。
- **タイトル**: 32-40pt。
- **本文**: 24pt（最低 18pt）。
- **配色**: 3色以内（意味を持たせる配色）。
