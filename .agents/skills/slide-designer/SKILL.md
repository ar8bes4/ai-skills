# 目的
- 聴衆が「読む」のではなく「見て理解する」スライドを制作するため、リサーチ結果を論理的かつ視認性の高い構成案に変換します。
- 医療系スライド特有のエビデンス提示、倫理的配慮、ターゲット（専門家/患者）別の構成ルールを厳守させます。

# この skill を使う条件
- 新しいプレゼンテーションの構成案を作成したいとき。
- リサーチ結果を整理し、Google Slides や Canvas に投入する前の「骨子」を作るとき。
- 作成済みのスライドに対し、デザイン上の改善アドバイスが欲しいとき。

# デザインの黄金律 (Core Rules)
本スキルは、以下の 5 つの原則に従ってアウトプットを生成します。

### 1. 構成：1スライド1メッセージ
- **ワンスライド・ワンメッセージ**: 1枚につき結論は1つ。2つある場合は分割。
- **タイトルとボディの連動**: タイトルだけで結論がわかるようにする。
- **「だから何？」を明示**: グラフや表の横には必ず結論（インサイト）を添える。

### 2. レイアウトと視覚整理
- **視線誘導**: Zの法則/Fの法則（左上から右下へ）を意識した配置。
- **整列と余白**: 要素の端を揃え、3〜5割の余白を確保して注目を惹きつける。

### 3. 文字とフォント
- **サイズ**: 本文 18pt 以上（推奨 24pt 以上）。
- **箇条書き**: 1枚 5〜7 行以内。体言止めを基本とする。

### 4. 色の制限 (3色ルール)
- **ベースカラー**: 背景/標準テキスト（白・黒・グレー）。
- **メインカラー**: 見出し/図形。
- **アクセントカラー**: 最重要箇所のみ（1箇所に絞る）。

### 5. ビジュアル
- **アイコンとシェーマ**: テキストだけの箇条書きを避け、視覚的な記号を併用する。

# 医療系スライド専門ルール
医療用資材を作成する場合は、以下の基準を強制的に適用します。

### 1. エビデンスと正確性
- **出典の明記**: 右下に著者名、雑誌名、発行年を小さく記載。
- **客観的表現**: 「～が示唆された」などデータに基づいた語尾。グラフの改変は厳禁。

### 2. プライバシーと倫理 (最重要)
- **個人情報の完全な匿名化**: 氏名、ID、日付の抽象化。
- **画像メタデータの加工**: CT/MRI等に写り込んだ個人情報を排除。

### 3. ターゲット別対応
- **学会用 (Pro)**: 効率重視。専門用語・略語・複雑なグラフも可。冷静なトーン。
- **患者用 (Patient)**: 安心感。中学生でもわかる言葉。イラスト・大きな文字。丁寧なトーン。

# 実行手順
1. **リサーチデータの入力**: 元となるテキストや論文要旨を受け取ります。
2. **ターゲットの特定**: 「医療従事者向け」か「患者向け」かを決定します。
3. **構成案の生成**: 
    - 各スライドのタイトルと結論（キーメッセージ）を定義。
    - Canvas に貼り付け可能な Markdown 形式で出力。
4. **デザイン指南**: 必要に応じて、使用すべきアイコンやグラフの簡略化案をアドバイス。

# 注意事項
- 文字を詰め込みすぎないよう、AI側で優先順位の低い情報を削る勇気を持ってください。
- 医療情報の最終的な正確性は常に制作者（医師）に確認を求める文言を添えてください。
