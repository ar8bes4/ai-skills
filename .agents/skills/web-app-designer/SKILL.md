# 目的
- リサーチ結果や専門的な情報を「見やすく、整った」ウェブサイト（SPA）として公開するため、洗練された UI/UX デザインをコードに落とし込みます。
- ユーザーにストレスを与えない「清潔感のあるデザイン（Minimalist/Clean Design）」を標準とします。

# この skill を使う条件
- 新しく SPA を作成したいとき。
- リサーチ結果をウェブサイト化して GitHub Pages で公開するフローのなかで、コード生成を担当するとき。

# デザイン黄金律 (Core Design Principles)

### 1. カラーパレット (Cleaning Palette)
- **ベースカラー**: `#f8f9fa` (オフホワイト) または `#ffffff` (白)。
- **メインテキスト**: `#1a1a2e` (深い紺) または `#2d3436` (濃いグレー)。
- **アクセントカラー**: `#0984e3` (ロイヤルブルー) をボタンや重要な見出しに使用。
- **余白**: 圧倒的な「白（White Space）」を確保し、情報を詰め込みすぎない。

### 2. タイポグラフィ
- **フォント**: `Inter`（Google Fonts）を推奨。
- **階層化**: 見出し (h1, h2, h3) のサイズ差を明確にし、視覚的な重みを付ける。
- **行間**: 本文は `line-height: 1.6 - 1.8` を確保し、読了率を高める。

### 3. レイアウト
- **カードレイアウト**: 各セクションを柔らかい影 (`box-shadow`) を持つ白いカードで囲み、情報の境界を明確にする。
- **ヒーローセクション**: 冒頭に主題を大きく提示し、一言で「何のサイトか」を伝える。
- **レスポンシブ**: モバイルファーストで設計し、デバイスを選ばず美しく表示。

### 4. インタラクションと可視化
- **Chart.js**: データを含むリサーチ結果の場合、グラフを積極的に活用して可視化する。
- **アニメーション**: `Intersection Observer` 等を利用した、スクロール時の控えめなフェードイン効果。

# 実行手順
1. **情報の受け取り**: リサーチされた事実、結論、データを受け取ります。
2. **構造設計**: 
    - ヘッダー（タイトル）
    - ヒーロー（要約）
    - 本文セクション（カード形式）
    - データセクション（グラフ/表）
    - フッター（出典/著者）
3. **コード生成**: 1つの HTML ファイル（CSS/JS 込み）にまとめ、即座にブラウザで動作する形式で出力します。

# 🚀 将来の拡張とセキュリティ (Vercel/Next.js 移行)
Gemini API 等のバックエンド連携を想定した設計指針を適用します。

### 1. セキュリティ（API キーの隠蔽）
- **厳禁**: クライアントサイドの JS に API キーを直書きしない。
- **対処**: Vercel 移行時には環境変数 (`.env`) を使用し、API Routes (Serverless Functions) 経由で API を叩く構成に変える。

### 2. フレームワークへの移行準備
- 現在生成している Vanilla JS 型のコードは、そのまま Next.js の `public` フォルダまたは `pages` フォルダに移植しやすい論理構造（コンポーネント指向）で構築する。
- デプロイ時には、将来的にビルド設定を自動認識させるための `package.json`（ダミーまたは初期構成）の生成を検討する。

# 注意事項
- 手垢のついた古いデザインを避け、現代的な「清潔感」を維持してください。
- 医療系、あるいは学術的な内容の場合、特に出典の記載を目立たせるように配置してください。
