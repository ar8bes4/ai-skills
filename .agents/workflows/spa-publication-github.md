このワークフローは、ディープリサーチから得られた大量の情報を、清潔感のあるデザインの SPA（Single Page Application）として構築し、既存のポートフォリオ用リポジトリ `yert1/hub1` のサブディレクトリに自動的にアップロード・公開するための手順です。

# 制作・公開プロセス

### ステップ①：ディープリサーチの実行
- **Antigravityの支援**: `search_web` を活用し、指定されたテーマの最新エビデンス、データ、トレンドを収集します。
- **目的**: 公開に耐えうる「質の高い情報」を整理すること。

### ステップ②：SPA コードの生成
- **Antigravityの支援**: `web-app-designer` スキルを活用し、HTML/CSS/JS が 1 つにまとまった、清潔感のあるデザインのコードを生成します。
- **デザイン方針**: 
    - 読みやすい Inter フォント、適切な余白、カードレイアウト。
    - リサーチ結果を要約したヒーローセクションと、詳細を語るセクションに分割。

### ステップ③：GitHub リポジトリ `research-output` への追加（デプロイ）
- **Antigravityの支援**: `remote-github_push_files` を使用して、指定されたディレクトリへファイルをプッシュします。
- **デプロイ先**: `ar8bes4/research-output` リポジトリの `research/[テーマ名]/index.html`。
- **注意点**: 既存のファイルを上書きしないよう、新しいフォルダ名を慎重に設定します。

### ステップ④：公開確認と URL 提示
- **最終チェック**: プッシュが完了したら、公開後の URL をユーザーに提供します。
- **公開URL形式**: `https://ar8bes4.github.io/research-output/research/[テーマ名]/`

---

# 推奨設定
- **ファイル名**: 常に `index.html` としてプッシュします。
- **リサーチテーマ**: フォルダ名は半角英数字（例: `medical-ai-2026`）を推奨します。
