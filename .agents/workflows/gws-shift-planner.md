# シフト作成自動化ワークフロー (`/gws-shift-planner`)

このワークフローは、`gws-shift-planner` スキルを活用して、複雑な当番シフトのパズルを解き、実務（カレンダー・メール）までを自動化します。

// turbo-all

1. **データロードと状況把握**
   - `gws-shift-planner` スキルを起動し、マスタスプレッドシート（ID: `1zIJ-ODc2noEqtJVMTUaGyDFuVIAuMlGV-2q0ykFr5r8`）から情報を取得します。
   - `基本ルール` シートから割当割合と禁止事項を、`希望入力` シートから不可日を取得します。

2. **シフト案の生成と提示**
   - 取得したデータを元に、AIが最適なシフト案を推論。
   - 変更履歴やコンフリクト（競合）の有無を含めた詳細なレポートをマークダウン形式で出力します。

3. **ヒューマンチェックと修正**
   - ユーザーに「○日の担当を変更して」といった対話的な修正を募ります。
   - 修正が完了し、ユーザーから承認（`y`）を得るまで待機します。

4. **一括反映アクション**
   - 承認後、以下の処理を連続実行します。
     - **カレンダー登録**: 指定のカレンダーIDへイベントを一括追加。
     - **報告下書き作成**: 指定の宛先（`byoin-kanri@city.iwata.lg.jp`）へ、シフト表を含んだメールの**下書き**を作成。

---
> [!TIP]
> **実行のコツ**: 「6月分のシフトを作って」のように時期を指定して呼び出すと、よりスムーズに開始できます。
