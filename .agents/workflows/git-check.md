# Git健康診断ワークフロー 2.0 (`/git-check`)

// turbo-all

このワークフローは、GitHub上のリポジトリとローカルPC（Windows）の状態を比較し、診断ミスを極限まで減らした正確なレポートを作成するためのものです。

### 厳密診断・同期ステップ (Strict Mode)

> [!IMPORTANT]
> **信頼性の保証**: 診断ミスを完全に排除するため、診断スクリプト内であらゆるリポジトリの `git fetch` を強制実行し、リモートの最新情報を取得してから比較を行います。

1. **GitHubリポジトリのリストアップ**
   - `remote-github_get_me` で自身のユーザー名を確認。
   - `remote-github_search_repositories` を使用して、自分が所有する全リポジトリを取得。

2. **専用診断スクリプトの実行**
   - `C:\Users\yert1\Documents\agy\00_System\ai-skills\scripts\git-health-check.ps1` を実行。
   - `$targetPath = "C:\Users\yert1\Documents\agy"` に対しての詳細なスキャン結果（ステータス、未コミット数、未プッシュ数、更新待ち数、リモートURL）を取得する。

3. **情報の統合と分析**
   - GitHub 側のリストとスクリプトの出力を統合し、以下のステータスに分類する。
     - **[未クローン]**: GitHubには存在するが、ローカルにはまだ落としていない。
     - **[変更あり]**: ローカルでファイルが編集されている（`MODIFIED` または `UNCOMMITTED_COUNT > 0`）。
     - **[未プッシュ]**: コミットはしたが、GitHubへ送信されていない（`UNPUSHED_COUNT > 0`）。
     - **[更新あり]**: GitHub側に新しいコミットがある（`BEHIND_COUNT > 0`）。
     - **[同期済み]**: すべてが最新（`CLEAN` かつ AHEAD/BEHIND ともに 0）。
     - **[ローカルのみ]**: ローカルにはあるが、GitHub側に同名のリポジトリがない（`REMOTE_URL` が `LOCAL_ONLY` または不一致）。
   - **アーカイブ済み**: GitHub上で `archived: true` のものは件数のみ報告（詳細診断はスキップ）。

### 報告形式

以下の項目を含むマークダウン・テーブルでレポートを作成してください。

| リポジトリ名 | 場所 | ステータス | 詳細内容（件数など） | 次のアクション案 |
| :--- | :--- | :--- | :--- | :--- |
| example-repo | GitHub | 未クローン | ローカルにありません | `git clone` しますか？ |
| my-project | Local | 変更あり | 3つのファイルが未コミット | コミットメッセージ案を作成しますか？ |
| api-tools | Local | 更新あり | リモートに 2 件の更新 | `fetch & diff` で内容を確認しますか？ |

> [!NOTE]
> テーブルの直後に、**「（外にアーカイブ済みリポジトリが N 件あります）」** と一行添えてください。

### 注意事項
- 常に **日本語** で親しみやすく報告すること。
- 必要に応じて、最も重要な「要対応」項目をテーブルの最上部に配置すること。
- **承認の要不要**: 診断は自動で行うが、コミットやプッシュ等の**変更を伴う操作**の前にのみ、必ず「y/n」で承認を得ること。
