# Git健康診断ワークフロー (`/git-check`)

このワークフローは、GitHub上のリポジトリとローカルPC（Windows）の状態を比較し、以下の項目を整理して報告するためのものです。

### 学習・診断ステップ

> [!NOTE]
> **自動実行の許可**: レポートの作成（ステップ1〜4）は全て読み取り専用の操作であるため、ユーザーへの「y/n」確認を待たずに一括で実行して報告すること。

1. **GitHubリポジトリのリストアップ**
   - `remote-github_get_me` で自身のユーザー名を確認。
   - `remote-github_search_repositories` を使用して、自分が所有する（またはアクセス可能な）全リポジトリのリストを取得。

2. **ローカルディレクトリの走査 (隠し属性対応)**
   - `C:\Users\yert1\Documents\agy` ディレクトリ全域を `-Force` オプションを付けて走査。
   - 隠し属性を持つ `.git` フォルダを確実に捕捉し、実体リポジトリをリストアップする。
   - ジャンクション（ReparsePoint）がある場合は、実体パスを特定した上で診断対象に含める。

3. **詳細ステータスの診断**
   - ローカルの各リポジトリに対して以下を実行：
     - `git status --short`: 未コミットの変更がないか。
     - `git cherry -v`: リモートにプッシュしていないコミットがないか。
     - `git remote -v`: 接続されているリモートURLを確認。

4. **情報の統合と分析**
   - GitHub側とローカル側を突き合わせ、以下のステータスに分類する。
     - **[未クローン]**: GitHubには存在するが、ローカルにはまだ落としていない。
     - **[変更あり]**: ローカルでファイルが編集されているが、コミットされていない。
     - **[未プッシュ]**: コミットはしたが、GitHubへプッシュ（反映）されていない。
     - **[同期済み]**: すべてが最新の状態。
     - **[ローカルのみ]**: ローカルにはあるが、GitHub側に同名のリポジトリが見当たらない。
   - **アーカイブ済み**: GitHub上で `archived: true` になっているリポジトリは詳細診断をスキップし、最後に件数のみをまとめて報告する。

### 報告形式

以下の項目を含むマークダウン・テーブルでレポートを作成してください。

| リポジトリ名 | 場所 | ステータス | 詳細内容（未コミット数など） | 次のアクション案 |
| :--- | :--- | :--- | :--- | :--- |
| example-repo | GitHub | 未クローン | ローカルにフォルダがありません | `git clone` しますか？ |
| my-project | Local | 変更あり | 3つのファイルが未コミット | コミットメッセージ案を作成しますか？ |

> [!NOTE]
> テーブルの直後に、**「（外にアーカイブ済みリポジトリが N 件あります）」** と一行添えてください。

### 注意事項
- 常に **日本語** で親しみやすく報告すること。
- リポジトリ数が多い場合は、特に「要対応」のものから優先的に表示すること。
- **承認の要不要**: 診断工程は自動で進めてよいが、「未コミットの変更内容に合わせて `README.md` を更新する」「一括デプロイ・プッシュする」等の**ファイル変更を伴う操作**の前にのみ、必ず「y/n」で承認を得ること。
