# Git 同期ワークフロー (`/pull-git`)

このワークフローは、特定のプロジェクト（または全リポジトリ）を最新の状態に同期するためのものです。

## 📥 実行方法
```bash
/pull-git [repo-name]
```
- **[repo-name] 指定あり**: 特定のリポジトリ（例: `obsidian`, `ai-skills`）のみをスキャンし、Pull（同期）します。
- **[repo-name] 指定なし**: `agy/` 配下の全カテゴリディレクトリ（00〜05およびObsidian）を対象に一括して最新化を試みます。

### ステップ①：リポジトリの走査と現状確認

> [!NOTE]
> このステップは自動で実行され、現在の状況を要約して報告します。

1. **リポジトリの特定**
   - **引数なし**: `C:\Users\yert1\Documents\agy` 以下の全カテゴリフォルダを走査し、Git プロジェクトを全てリストアップします。
   - **[repo-name] あり**: 指定したキーワード（例：`obsidian`, `ai-skill`）を含むリポジトリのみを対象にします。

2. **状態のチェック**
   - 各リポジトリに対して以下を実行します。
     - `git status --short`: 未コミットの変更がないか（Dirty状態の確認）。
     - `git fetch`: リモートの最新情報を取得。
     - `git rev-list --count HEAD..@{u}`: Pullすべきコミットがいくつあるか。

3. **診断レポートの提示**
   - 以下の形式で現在の状況を報告し、一括実行の承認を得ます。

| リポジトリ名 | 状態 (Dirty) | Pull対象件数 | 備考 |
| :--- | :--- | :--- | :--- |
| ObsidianVault | Clean | 3 commits | Pull可能です |
| ai-skills | **Dirty (3 files)** | 0 commits | **コンフリクトの恐れあり** |

### ステップ②：一括Pullの実行

1. **ユーザー確認**
   - **[ユーザー確認]** 「Cleanなリポジトリに対して一括Pullを実行しますか？ (y/n)」
   - Dirtyなリポジトリについては、個別にスキップするか、または先に解消することを推奨します。

2. **Pullの実行**
   - 承認が得られたリポジトリに対し、`git pull` を順番に実行します。

### ステップ③：完了報告

- 実行結果（成功/失敗）をまとめ、「全ての同期が完了しました」または「一部にエラーが発生しました」と報告します。
