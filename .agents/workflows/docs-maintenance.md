# ドキュメント整備ワークフロー (`/docs-maintenance`)

このワークフローは、リポジトリが拡張されるたびに、常にドキュメントが最新かつ「日本語ルール」に従った状態であることを保証するためのものです。

### 🚨 診断工程 (全自動)

> [!NOTE]
> **自動実行の許可**: ステップ1〜3は情報の読み取りと分析のみであるため、ユーザーの「y/n」確認を待たずに一括で実行して報告すること。

1.  **スキルとワークフローの目録作成 (Inventory)**
    - `.agents/skills/` および `.agents/workflows/` 以下を走査し、実在するフォルダ・ファイル名をリストアップ。
    - 各 `SKILL.md` のフロントマター `description` を抽出。

2.  **README の整合性確認 (Consistency Check)**
    - ルート `README.md` および各ディレクトリの `README.md` を読み取る。
    - 目録（Inventory）にあるがREADMEに未記載の項目、またはREADMEにあるが実在しない項目を特定。

3.  **日本語記述と内容のチェック (Validation)**
    - `SKILL.md` の `description` が日本語でない、または極端に短い（初期状態のまま）ものを抽出。
    - 各README内のリンクが正しい絶対/相対パスになっているか確認。

### 📋 診断レポートの提示

以下の内容を含むレポート（マークダウン形式）を表示してください。

| カテゴリ | 項目名 | チェック結果 | 必要なアクション |
| :--- | :--- | :--- | :--- |
| スキル | `new-awesome-gem` | README 未記載 | `.agents/skills/README.md` への追加 |
| スキル | `old-tool` | Description が英語 | `SKILL.md` の日本語化 |
| ワークフロー | `/new-task` | ルート未記載 | `README.md` への追加リンク |

---

### 🛠 整備工程 (要承認)

診断レポートを提示した後、**「これらの修正案を一括で反映しますか？ (y/n)」** とユーザーに確認してください。

1.  **README の自動更新**:
    - 分類に基づき、適切なテーブル位置に不足項目を挿入。
2.  **SKILL.md の自動翻訳 / 更新**:
    - AI の推論により最適な日本語 `description` を作成し反映。
3.  **完了報告**:
    - 全ての修正が完了したことを通知し、必要に応じて `git-push` を提案。

### 注意事項
- 全てのドキュメントは **日本語** で記述すること（最上位命令の遵守）。
- 技術的なパラメータやコマンド名自体は改変せず、解説部分のみを日本語化すること。
- **承認の徹底**: ファイルの書き換え（`write_to_file`, `replace_file_content`）を行う直前には、必ず最終確認（y/n）を得ること。
