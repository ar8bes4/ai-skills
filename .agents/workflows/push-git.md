# Git 同期ワークフロー (`/push-git`)

このワークフローは、特定のプロジェクト（または全リポジトリ）の変更を安全に GitHub へ反映させるためのものです。

## 📥 実行方法
```bash
/push-git [repo-name]
```
- **[repo-name] 指定あり**: 特定のリポジトリ（例: `obsidian`, `ai-skills`）のみをスキャンし、同期します。
- **[repo-name] 指定なし**: `agy/` 配下の全カテゴリディレクトリ（00〜05およびObsidian）を対象に一斉スキャンを実施します。

### ステップ①：変更箇所の抽出

> [!NOTE]
> このステップは自動で実行され、変更のあるリポジトリを要約します。

1. **リポジトリの走査**
   - **引数なし**: `C:\Users\yert1\Documents\agy` 以下の全カテゴリフォルダを走査します。
   - **[repo-name] あり**: 指定された文字列を含むディレクトリのみを検索対象とします。

2. **変更のチェック**
   - 各リポジトリに対して以下を実行します。
     - `git status --short`: 未コミットの変更ファイル。
     - `git cherry -v`: 未プッシュのコミット。

3. **要約レポートの提示**
   - 以下のように「手を入れたリポジトリ」を一覧表示します。

| リポジトリ名 | 未コミット数 | 未プッシュ件数 | 備考 |
| :--- | :--- | :--- | :--- |
| ai-skills | 2 files | 0 | コミットが必要です |
| ObsidianVault | Clean | 1 commit | プッシュのみ可能 |

### ステップ②：コミット・プッシュ内容の確認

1. **一括処理の承認**
   - **[ユーザー確認]** 「変更のあるリポジトリを順番に処理しますか？ (y/n)」

2. **リポジトリごとの個別メッセージ提案**
   - **[Antigravity の動作]** 変更のある各リポジトリに対し、`git diff` を分析して**最適なコミットメッセージを提案**します。
   - **[ユーザー確認]** 提案されたメッセージで良いか、または修正が必要かを順番に確認します。
   - メッセージが確定したリポジトリから順次、コミットとプッシュを実行します。

3. **実行**
   - 確定されたメッセージで以下の処理を行います。
     - `git add .`
     - `git commit -m "<提案・確定したメッセージ>"`
     - `git push origin HEAD`

### ステップ③：完了報告

- 実行後のステータスをまとめ、「全ての変更が GitHub に反映されました」と報告します。
- 必要に応じて、GitHub の各リポジトリ URL を添えます。
