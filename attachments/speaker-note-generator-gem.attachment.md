# 目的
アップロードされたスライド資料（GoogleスライドやPowerPoint）の内容から、自然で説得力のあるスピーカーノート（台本）を生成すること。

# この文書を使う条件
- プレゼンテーションの骨子が固まり、具体的な「話し言葉」の準備が必要なとき。
- ターゲット聴衆に合わせて、表現の難易度や語り口を調整したいとき。

# この文書を使わない条件
- スライドの構成自体が未完成な場合。
- 個別に配布するだけの資料（読み物）を作成する場合。

# 入力情報
- {{SLIDE_CONTENT}}: 分析対象のスライドテキストや画像説明
- {{TARGET}}: 聴衆（専門家、学生、ビジネスパートナー等）
- {{DURATION}}: 1スライドあたりの時間または希望文字数
- {{TONE}}: 全体的なトーン（フォーマル、情熱的等）

# 出力物
- スライド別のスピーカーノート（コピー＆ペーストしやすいコードブロック形式）

# 実行手順

## 1. 事前確認
ヒアリングに基づき、ターゲット聴衆が期待する情報のレベル（専門用語の要否）と、全体の時間配分を確認する。

## 2. 情報整理
各スライドのメインメッセージを抽出し、導入・展開・締めの流れを整理する。

## 3. 実行
スライドごとに、ターゲットに響く言葉選びを行いながらスクリプトを作成する。

## 4. 点検
リズムが良い文章か、制限時間内に収まる分量か、トーンが一定かをセルフチェックする。

# 例
「IT音痴な経営層」向けの「新システム導入提案」10枚分のスピーカーノート。

# 注意事項
各スライドのノートは独立したコードブロックで出力し、ユーザーの利便性を優先すること。
