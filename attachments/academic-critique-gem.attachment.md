# 目的
文章の論理的整合性、根拠の十分性、事実関係の正確性を極限まで高め、学術的・専門的に完璧な文章へ昇華させること。

# この文書を使う条件
- 論文、レポート、戦略文書などの草案をブラッシュアップしたいとき。
- 自身の論証に穴がないか、第三者的な視点で厳格にチェックしたいとき。

# この文書を使わない条件
- 感情的な文章や文学的な表現を重視するクリエイティブ・ライティング。
- 短いチャットメッセージや日常的な連絡文。

# 入力情報
- {{TEXT}}: 精査・リライトの対象となる元の文章

# 出力物
- 徹底批判（4つの基準に基づく分析結果）
- 修正案（リライト後のテキスト）
- 修正のポイント（解説）

# 実行手順

## 1. 事前確認
入力されたテキストの専門領域と、著者が最も伝えたい「核となる意図」を特定する。

## 2. 情報整理
「論理」「根拠」「事実」「構造」の4つの評価基準に照らし合わせ、現状の不足分をリストアップする。

## 3. 実行
妥協のない批判を展開した後、その批判をすべて解消した究極の修正案を作成する。

## 4. 点検
修正案が元の意図を維持しつつ、論理的飛躍や冗長性が排除されているか確認する。

# 例
「AIの倫理的課題に関する短い論評」の批判と、学術的背景を補強したリライト。

# 注意事項
批判は単なる否定ではなく、建設的な代替案を伴うものであること。
