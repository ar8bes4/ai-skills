# 目的
ユーザーの入力に含まれる情報不足（Gap）を特定・補完し、LLM が最高精度で実行できる XML 形式のプロンプトを設計すること。

# この文書を使う条件
- 新しい AI アシスタントや特定のタスク用プロンプトをゼロから作成したいとき。
- 既存のプロンプトの精度が低く、構造的な改善が必要なとき。

# この文書を使わない条件
- 既に完成されたプロンプトの微調整のみを行う場合。
- 非常に単純なワンショットの質問。

# 入力情報
- {{USER_REQUEST}}: プロンプト化したいユーザーの要求・目的。

# 出力物
- 前提条件の確認（不足要素、本来の質問、適用した仮定の表）
- 最適化されたプロンプト（XML 構造化プロンプト）

# 実行手順

## 1. 事前確認
ユーザー入力から「役割」「背景」「具体的指示」「出力形式」の欠落を分析する。

## 2. 情報整理
欠落している情報に対し、最もらしい「回答例（仮定）」を作成し、表形式で可視化する。

## 3. 実行
仮定した情報を反映させ、`<role>`, `<context>`, `<instructions>`, `<output_format>` タグを用いた XML プロンプトを構築する。

## 4. 点検
出力形式の強制（Markdown 形式）が含まれているか、論理構造が明確かを確認する。

# 例
「ブログ記事を書いて」という要求に対し、対象読者やトーンを仮定して XML プロンプトを生成する例。

# 注意事項
生成された XML プロンプトの `<context>` タグ等は、ユーザーが後で容易に修正できるように設計すること。
